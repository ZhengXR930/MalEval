# Security and Responsible Handling

MalEval concerns Android malware analysis. Some files in the associated
dataset may contain malicious code.

- Never install or execute APK samples on personal or production devices.
- Use a disposable, isolated research environment with restricted networking
  and no access to sensitive credentials or data.
- Treat all extracted files and generated payloads as untrusted.
- Follow your institution's malware-handling policy and all applicable laws
  and third-party terms.
- LLM stages can transmit prompts or code excerpts to external providers.
  Review provider policies and avoid sending material you are not authorized
  to disclose.

For questions about the artifact, use the GitHub issue tracker. Do not attach
malware samples to public issues.
